## ⚠️ Important Reminders

### Before Committing Code

- [ ] Verify `.env` is NOT staged for commit
- [ ] Verify `google-services.json` is NOT staged for commit
- [ ] Check that no API keys are hardcoded in any files
- [ ] Ensure `.env.example` has placeholder values only

### Before Sharing Code

- [ ] Share `.env.example` (safe to share)
- [ ] Share actual credentials through secure channels only
- [ ] Never share credentials via email, Slack, or public channels
- [ ] Use password managers or secure vaults for credential sharing

### Regular Maintenance

- [ ] Rotate API keys every 90 days
- [ ] Review who has access to credentials
- [ ] Check for any exposed keys in commit history
- [ ] Update `.env.example` when adding new variables

## 🔍 How to Check for Exposed Secrets

### Check Git Status
```bash
git status
```
Make sure `.env` is not listed in "Changes to be committed"

### Check Staged Files
```bash
git diff --cached
```
Verify no API keys are visible

### Search for Hardcoded Keys
```bash
# Search for potential API keys in code
grep -r "AIza" lib/
grep -r "api_key" lib/
```

## 🚨 If You Accidentally Commit Secrets

1. **Immediately rotate the exposed keys** in Firebase Console
2. Remove the file from Git history:
   ```bash
   git filter-branch --force --index-filter \
   "git rm --cached --ignore-unmatch .env" \
   --prune-empty --tag-name-filter cat -- --all
   ```
3. Force push (coordinate with team first):
   ```bash
   git push origin --force --all
   ```
4. Update all team members to pull the cleaned history

## 📝 Environment Variables Reference

Current variables in use:

- `FIREBASE_ANDROID_API_KEY` - Firebase Android API Key
- `FIREBASE_ANDROID_APP_ID` - Firebase Android App ID
- `FIREBASE_ANDROID_MESSAGING_SENDER_ID` - Firebase Messaging Sender ID
- `FIREBASE_PROJECT_ID` - Firebase Project ID
- `FIREBASE_STORAGE_BUCKET` - Firebase Storage Bucket
